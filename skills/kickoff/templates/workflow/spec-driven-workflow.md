## Workflow: Spec-Driven

1. **Define spec**: Write or review the formal specification/contract
2. **Validate spec**: Ensure spec is complete and consistent
3. **Implement to spec**: Code must satisfy the specification exactly
4. **Verify against spec**: Automated checks against the spec
5. **Document deviations**: If any deviation is needed, document why

### Rules
- The spec is the source of truth
- Never deviate from spec without explicit human approval
- Spec changes require a separate discussion before implementation
