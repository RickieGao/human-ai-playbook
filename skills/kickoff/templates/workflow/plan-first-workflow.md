## Workflow: Plan First

1. **Explore**: Read relevant source files, understand current architecture
2. **Plan**: Propose a detailed implementation plan with steps
3. **Review**: Wait for human approval of the plan
4. **Execute**: Implement step by step, commit after each step
5. **Verify**: Run tests and present results for review

### Rules
- Never start coding before the plan is approved
- If the plan needs to change during execution, stop and discuss
- Each plan step should be independently verifiable
