## Workflow: Incremental

1. **Start small**: Pick the smallest useful increment
2. **Implement**: Build it quickly
3. **Verify**: Test and review
4. **Iterate**: Based on feedback, pick the next increment
5. **Refactor**: Clean up accumulated rough edges periodically

### Rules
- Each increment must be shippable (no half-done features)
- Keep increments under 1 hour of work
- Refactoring is a separate increment, not mixed with features
