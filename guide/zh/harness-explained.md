# Harness 是什么

> `/kickoff` 会为你生成一套完整的 harness 配置。这里解释它包含什么、为什么。

---

## Harness = AI 的工作环境配置

就像新员工入职需要配置电脑、了解公司规范一样，AI 在每个项目里也需要知道：这个项目是什么、你的工作方式、什么能做什么不能做。Harness 就是这套配置的总称。

---

## 五个组成层

### 1. 上下文注入
**是什么**：`CLAUDE.md` + 规则文件
**作用**：告诉 AI 项目背景、协作方式、质量标准
**示例**：`这是一个 Python FastAPI 项目，提交前必须跑测试`

### 2. 行为约束与自动化
**是什么**：`settings.json` + hooks 配置
**作用**：设定 AI 的权限边界，自动触发某些行为
**示例**：`编辑文件后自动运行 lint`，`禁止直接 push 到 main`

### 3. 可复用工作流
**是什么**：`.claude/skills/` 下的自定义命令
**作用**：把常用操作封装成一键命令
**示例**：`/commit` 自动规范化 commit message，`/review-pr` 代码审查

### 4. 外部能力扩展
**是什么**：MCP server 配置
**作用**：连接外部工具（GitHub、数据库、文档系统等）
**示例**：AI 直接读写 GitHub Issues，查询数据库 schema

### 5. 演化机制
**是什么**：定期更新 harness 配置的流程
**作用**：harness 随着使用经验持续改进
**示例**：发现某个规则没用 → 反馈 → 更新模板

---

## 个人 Profile

你的核心偏好（协作风格、常用工具、安全红线）存在 `~/.claude/harness-profile.md`，跨所有项目共享。`/kickoff` 读取它，加上项目特异配置，合并生成当前项目的完整 harness。

新项目部署时，90% 的配置来自你的 profile，只需回答少量项目特定问题。
