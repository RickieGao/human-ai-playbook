- Explain "why" for non-obvious choices, skip the "what".

### Autonomy
- Execute plan steps independently, pause at checkpoints for review.

- For new features: plan first, get approval, then execute.

## Workflow: Plan First

1. Explore relevant files before proposing changes
2. Present a plan with numbered steps
3. Wait for my approval
4. Execute step by step, commit after each
5. Run tests and show results

## Quality Standards

- All new features must have tests
- Run `npm test` before committing
- Use conventional commits
- Never skip pre-commit hooks
